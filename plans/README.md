## Deferred — valid findings NOT planned in this batch
(Recorded so they are not re-audited; promote to a plan when prioritized.)

- **Unvalidated config/plugin import** (`MyDialogFragment.java:192-242`): imported
  `config.json` silently redirects the server; no size cap (zip-bomb); null-deref;
  stream leak. HIGH confidence, M effort. Strong candidate for the next batch;
  partly mitigated once 001 lands (zip-slip), but validation/size-cap still missing.
- **Swallowed first-run unpack failures** (`AsyncAssetManager.java:57-97`): reports

  `ProgressRepository`.
- **MainActivity static View leaks** (`MainActivity.java:62-76`, not nulled in
  `onDestroy`): Activity/GL-surface retained. HIGH, M; MED risk (static access from
  native callbacks).
- **`ProgressKeeper.waitUntilDone` CME** (`ProgressKeeper.java:39-44` vs `:94`):
  list mutated mid-iteration. MED, S. Fix: iterate a copy in `updateTaskCount`.
- **`downloadFile` dead `bos` var / leak on failure / unchecked `renameTo`**
  (`DownloadUtils.java:56-90`): S. Natural to fold into 002's follow-up.
- **`System.gc()` on the UI thread ×3** (`MainActivity.java:392,396,494`): S, LOW.
- **tar-slip + unchecked symlink** (`MultiRTUtils.java:242,250`): MED — same class as
  001 but for runtime tarballs; exploitability needs a caller check.
- **`allowBackup="true"`** (`AndroidManifest.xml:23`): LOW, S.
- **Shared 4-thread executor misconfigured** (`PojavApplication.java:27`): MED, M.
- **`Tools.java` god object / `MainActivity` mixing concerns**: L, tech-debt — defer

## Findings considered and rejected (not real issues)
- `ProgressRepository` listeners never removed — **not a leak**: owned by the
  process-lifetime `AppContainer`; strong refs are intentional.
- `ScapeLauncher` progress listener — correctly balanced (add in `onCreate`, remove
  in `onDestroy`).
- Compose/`StateFlow` threading — correct (thread-safe `MutableStateFlow` +
  `collectAsStateWithLifecycle`); no cross-thread Compose mutation.
- `res/xml/provider.xml` broad root path — latent only; no `<provider>` is wired to
  it, so nothing is exposed today (tighten before any FileProvider is added).
- Debug keystore password in `build.gradle` — standard Android debug keystore,
  by-design, not a secret.
