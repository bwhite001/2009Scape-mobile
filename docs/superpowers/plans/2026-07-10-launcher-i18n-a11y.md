# Launcher i18n + Accessibility Bundle — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the RS2-reskinned launcher's hardcoded English strings into `strings.xml` (restoring Crowdin translation), and bring its Compose controls up to Android's 48dp touch-target and TalkBack accessibility baseline.

**Architecture:** Presentation-only. Preference metadata switches from a resolved `String` label to a `@StringRes Int` resolved at render time; home/settings screens reference `stringResource`; toasts use `getString(resId, arg)`. Accessibility is added via `Modifier.minimumInteractiveComponentSize()` and `semantics {}` on the shared `ui/rs` composables. No behavior, logic, or SharedPreferences key changes.

**Tech Stack:** Kotlin, Jetpack Compose (Material 3), Android resources, ktlint.

**Spec:** `docs/superpowers/specs/2026-07-10-launcher-i18n-a11y-design.md` (string key inventory is the Appendix there and is reproduced in Task 1).

## Global Constraints

- **No logic/behavior changes.** SharedPreferences keys (`ignoreNotch`, `haptic`, etc.) are untouched — only the human-readable labels move to resources.
- **English source only.** Edit `app_pojavlauncher/src/main/res/values/strings.xml` only. Do NOT add or edit any `values-*/` locale. Do NOT run `scripts/languagelist_updater.sh` (no locale added). Crowdin ingests new source keys automatically.
- **Naming:** `snake_case`, matching existing `strings.xml`. Prefixes: `launcher_*`, `settings_*`, `pref_*`.
- **"2009Scape" stays a literal** (brand/proper noun). Everything else user-facing is a resource.
- **Per-task verification gate** (the "build gate"), run via the checked-in Docker image (no host JDK/SDK):
  ```bash
  docker run --rm -v /runescape/2009Scape-mobile:/project 2009scape-apk-builder \
    bash -lc "cd /project && ./gradlew :app_pojavlauncher:ktlintCheck :app_pojavlauncher:assembleDebug --stacktrace"
  ```
  Expected: `BUILD SUCCESSFUL`. `ktlintCheck` is Kotlin-only; the strings-only task (Task 1) still runs `assembleDebug` because aapt validates `strings.xml`.
- **Deferred on-device checks** (cannot run headless — recorded per `docs/verification/device-smoke-checklist.md`, performed at end, not per task): translated locale renders; TalkBack announces roles/values and skips decorations; all controls tappable at 48dp.
- **Commit style:** conventional commits. Do NOT add a `Co-authored-by: Copilot` trailer.
